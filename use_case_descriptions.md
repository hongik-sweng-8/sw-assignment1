### Use case description: 자전거 대여

| Actor Action                                                    | System Response                                           |
|----------------------------------------------------------------|----------------------------------------------------------|
| 1. 회원이 '대여 정보 조회' 버튼을 클릭한다.| 2. 현재 대여 중인 자전거 정보(대여소 이름, 위치, ID, 제품명, 유형)가 화면에 출력된다.|

Extensions:  
- After step 2,  대여 반납

| Actor Action              | System Response             | External Actor Action(이메일 전송서비스)     | External Actor Action( 요금 결제 서비스)    |
|--------------------------|-------------------------|-------------------------------|------------------------|
| 3. 회원이 자전거 반납 버튼을 클릭한다.| 4. 선택된 자전거가 반납되었음을 알리는 메시지를 출력한다 | 5. 반납된 자전거에 대기 예약한 회원이 있는 경우 대기 1순위 회원에게 예약 확정 이메일을 전송한다.| 6. 사용 시간에 따라 요금이 자동 결제된다  | 


Extensions:  
- After step 4, 식당 추천
  
| Actor Action                                          | System Response   | External Actor Action(외부 식당 추천 서비스)   |
|-----------------------------------------------|---------------------|----------------------------------------------------------|
| 5. 회원이 '근처 식당 추천' 버튼을 클릭한다.| 6. 외부 식당 추천 서비스로 연결한다. | 7. 사용자 위치 정보를 기반으로 근처 식당을 화면에 띄운다. |

External &&  system Extensions(외부 시스템이므로 포함되지 않음):  
- After step 7, 식당 예약
  
| Actor Action                                     | System Response   | External Actor Action(외부 식당 추천 서비스)   |
|-----------------------------------------------|---------------------|----------------------------------------------------------|
| 8. 회원이 추천 식당 중 하나를 선택하고 '예약' 버튼을 클릭한다.| NONE | 9. 예약 확정을 화면에 띄운다. |
